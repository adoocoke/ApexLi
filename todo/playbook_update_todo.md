# Trading Playbook v3 更新与优化 TODO

> 目标：将当前 Playbook 从“经验总结文档”升级为更适合 **LangGraph Agent** 使用的结构化、可执行知识体系。

## 整体评价

当前 Playbook 有较好的交易理念和方法论基础（尤其是量仓分析和交易者行为认知），但在**结构化程度、规则可执行性、Few-shot 质量**上存在明显不足，导致 Agent 难以稳定、高质量地使用。

---

## 优先级改进计划

### P0（最高优先级 - 强烈建议优先完成）

#### 1. Prompt 结构化与模块化
- **问题**：当前 Playbook 以长文本形式存在，LLM 难以精准引用和执行。
- **目标**：将 Playbook 拆分为结构化模块，便于 System Prompt、RAG 检索和 Tool 调用。
- **行动建议**：
  - 将 Playbook 拆分为以下模块：
    - `Core_Principles.md`（核心交易理念）
    - `Volume_Position_Analysis.md`（量仓分析核心逻辑）
    - `Trend_and_System_Selection.md`（趋势与系统选择）
    - `Behavior_Guidelines.md`（交易者行为规范）
  - 每个模块输出结构化 JSON Schema（字段 + 描述 + 示例）
- **预期收益**：大幅提升 Agent 对 Playbook 的遵循度和可解释性。

#### 2. Few-shot 示例重构
- **问题**：现有示例质量参差不齐，Thought 部分逻辑不够严谨，缺少反例。
- **目标**：打造高质量、逻辑清晰的 Few-shot 示例集。
- **行动建议**：
  - 对现有 6 个示例进行精炼和重写
  - 增加 3~4 个**反面示例**（Negative Examples）
  - 每个示例严格遵循：Observation → Thought（逻辑链条）→ Action
  - 重点强化“没有定式不交易”“主动放弃”等核心理念
- **预期收益**：显著提升 Agent 的推理质量和决策一致性。

#### 3. 量仓分析 Tool 化（最高价值）
- **问题**：量仓分析（2.1）是 Playbook 最核心的内容，但目前仅靠 LLM 自由判断，稳定性差。
- **目标**：将量仓核心逻辑做成可被 Agent 调用的结构化 Tool。
- **行动建议**：
  - 设计 `analyze_volume_position` Tool
  - 输入：价格变化、持仓变化、成交量变化、是否在关键位置、趋势方向
  - 输出：结构化结论 + 置信度 + 引用规则
- **预期收益**：把 Playbook 中最强的武器从“描述”变成“可执行能力”。

### P1（重要，建议在 P0 之后推进）

#### 4. 增加主动放弃机制（Reflection Node）
- **问题**：当前 Agent 缺少“该不该继续分析/交易”的自我评估能力。
- **目标**：让 Agent 能够主动判断是否应该放弃当前机会。
- **行动建议**：
  - 在 Graph 中增加 `Reflection` 节点
  - 依据 Playbook 中“何时主动放弃”的规则进行判断
  - 输出是否继续 + 放弃理由

#### 5. 定义规则优先级与冲突解决机制
- **问题**：当多个规则出现冲突时，Agent 没有明确的判断依据。
- **目标**：建立规则优先级体系。
- **行动建议**：
  - 给核心规则划分优先级（如：量仓 > 趋势 > 形态）
  - 在 Prompt 中增加冲突解决指引

### P2（中长期优化）

#### 6. 行为轨迹人格化注入
- 将第3节（行为轨迹）内容浓缩后，作为 Agent 的“交易者人格”注入 System Prompt。

#### 7. 领域模型支持
- 配合 Pydantic 模型，对 Observation、Signal 等核心概念进行结构化定义，减少自由文本带来的不确定性。

## 执行建议顺序

1. **第一阶段**（优先级最高）
   - Prompt 结构化 + 模块化
   - Few-shot 示例重构

2. **第二阶段**
   - 增加 Reflection / 主动放弃节点
   - 定义规则优先级

3. **第三阶段**
   - 行为轨迹人格化
   - 领域模型建设
